# AI 의회 로그: 2026-04-18

## 안건 1: 1팀 5인 1포지션 원칙 보장 및 UI 증식 버그 해결

### 1. 질문 구성 (Problem Definition)
- **맥락**: 인게임 정보 조회 시 상대팀(Enemy Team)의 포지션이 정글 2명으로 중복 할당되거나, 수동 수정 시 카드 항목이 5개에서 7개로 늘어나는 현상 발생.
- **결정**: 기존의 '정글러 선제외' 소거법을 폐기하고 새로운 배정 아키텍처 도입 여부.
- **제약 조건**: 반드시 한 팀에 탑, 정글, 미드, 원딜, 서폿이 각각 1명씩 배정되어야 함. 어떤 예외 상황(강타 2명 등)에서도 1:1 매칭이 유지되어야 함.

### 2. 관점 수집 (AI Perspectives)

#### **Gemini (Technical Efficiency)**
- **권고**: 5인 전원을 대상으로 한 5! (120가지) 완전 탐색 백트래킹 도입.
- **이유**: 120가지 경우의 수는 연산량이 매우 적으며, 수학적으로 중복 배정이 원천 봉쇄됨.
- **위험**: 강타가 없는 팀의 경우 정글 배정이 모호해질 수 있으나, 가중치 시스템으로 해결 가능.

#### **Codex (Implementation & Stability)**
- **권고**: `scorePlayerForLane` 함수에 '강타 전용 가중치'를 부여하고 프론트엔드 `applyLaneSwap`의 무결성 강화.
- **이유**: 백엔드에서 1:1 매칭이 보장된 데이터가 내려오면 프론트엔드의 상태 불일치 버그가 자연스럽게 해결됨.
- **위험**: 상태 업데이트 시 배열 불변성을 지키지 않으면 여전히 UI 렌더링 에러가 발생할 수 있음.

#### **Claude (Correctness & Minimal Blast Radius)**
- **근본 원인 재진단**: `buildRoleMap`에서 스마이트 감지 시 정글러를 미리 제외하는 로직이 스마이트 0명 또는 2명 이상인 상황에서 역할 뒤바뀜을 초래함. 또한 `clientRescore.ts`에서 동일 라인 점유 시 조용히 덮어쓰는 버그 식별.
- **권고**: `LaneKey | null` 슬롯을 이용한 5! 백트래킹으로 아키텍처를 단순화하되, 기존 타입 컨벤션(`null = 정글`)을 유지하여 파급 범위를 최소화할 것.

### 3. 종합 및 심의
- **합의 영역**: 5인 통합 백트래킹이 가장 확실한 해결책이라는 점에 만장일치.
- **불일치 영역**: 없음.
- **고유 통찰**: "정글러를 미리 제외하는 행위가 모든 버그의 시작점이었다"는 근본 원인 식별 및 `clientRescore.ts` 덮어쓰기 결함 발견.

### 4. 최종 결정 및 근거
- **결정**: **[5인 통합 라인 배정 아키텍처]**로 전면 개편.
- **근거**: 수학적 무결성 확보, 변칙 메타(강타 2명 등) 대응력 강화, 데이터 구조적 정합성을 통한 UI 증식 버그 차단.

## 안건 2: 뉴메타(비원딜/봇파괴) 대응 및 라인 배정 적중률 95% 달성 전략

### 1. 질문 구성 (Problem Definition)
- **맥락**: 세나-사이온 같은 비정석 조합이나 비원딜 메타에서 라인 오판이 발생하면 정글 동선 추천 시스템의 신뢰도가 붕괴됨.
- **결정**: 보편적 데이터(스펠/통계)와 예외적 데이터(DNA/시너지)를 어떤 계층 구조로 결합하여 95% 이상의 정확도를 확보할 것인가?
- **제약 조건**: 1팀 5인 1포지션 원칙 준수, 보편적 상황의 안정성 유지.

### 2. 관점 수집 (AI Perspectives)

#### **Gemini (Data-Driven Anchor)**
- **권고**: **'플레이어 DNA(최근 전적)'**를 최우선 닻(Anchor)으로 설정. 최근 10판의 데이터 내에서 유저의 포지션 습관은 90% 이상 일관성을 보임.
- **위험**: 부캐나 급격한 포지션 변경 유저는 놓칠 수 있으나, 최근 판에 가중치를 주는 `Time Decay`로 보완 가능.

#### **Codex (Logic & Heuristics)**
- **권고**: **'배타적 스펠/룬 시그니처'**와 **'듀오 시너지 사전'**을 통한 소거법 강화. 정화/배리어는 라이너, 수호자/빙결은 서포터의 강력한 지표임.

#### **Claude (Hierarchical Orchestrator)**
- **권고**: **[4단계 계층적 검증 시스템]** 도입.
    1. Layer 1: 강타(정글) 고정.
    2. Layer 2: 독점적 신호(정화/룬)로 그룹핑.
    3. Layer 3: 듀오 시너지(세나 등) 기반 파트너 매칭.
    4. Layer 4: 5! 백트래킹 글로벌 최적화.

#### **Claude 상세 입장 (코드 분석 기반)**

실제 코드를 읽고 나서 세 가지를 말할 수 있다.

**1. 현재 구현 상태 재평가 — 이미 대부분 완성됨**

4단계 계층 구조는 이미 구현되어 있다:
- Layer 1 (강타): ✅ `hasSmite` 감지
- Layer 2 (정화/배리어 하드락): ✅ Phase 2에서 구현 완료
- Layer 3 (봇 듀오 시너지): ✅ `findBotDuo` + `bot-meta-rules.json`
- Layer 4 (5! 백트래킹): ✅ 안건 1에서 구현 완료

룬 가중치도 `getPerkPositionalWeights()`에서 키스톤별로 0.05~0.30 범위로 세밀하게 구현되어 있다. 남은 문제는 아키텍처가 아니라 **가중치 공식의 취약 구간**이다.

**2. 실제 90% → 95% 갭의 정체**

현재 공식: `DNA×0.60 + champWeight×0.30 + spellRune×0.10`

DNA 60% 가중치는 샘플이 충분할 때(5판 이상) 정확하다. 그러나 샘플 부족 구간에서 공식이 무너진다:

```
예시: 신규 유저 / 부캐 — sampleSize = 1판
DNA supportRate = 1.0 (딱 1판 서폿 플레이)
→ dnaWeight = 1.0 × 0.60 = 0.60  (과신)

예시: 포지션 전환 중인 플레이어 — 최근 7판 ADC, 3판 서폿
DNA supportRate = 0.30 (시간 감쇠 후)
→ dnaWeight = 0.30 × 0.60 = 0.18  (과소)
```

즉, DNA가 60%를 지배하는 현재 구조는 **샘플이 많을 때는 강하지만, 샘플 경계 구간(1~3판)에서 노이즈를 증폭**한다. 이것이 90%→95% 갭의 주원인이다.

**3. 권고: Adaptive Signal Fusion (적응형 신호 융합)**

가중치를 `sampleSize`에 따라 동적으로 조정한다:

```
dnaTrust = min(1.0, sampleSize / 5)  // 0판→0.0, 3판→0.6, 5판→1.0

dnaW     = 0.20 + dnaTrust × 0.40   // 0.20 → 0.60
champW   = 0.50 - dnaTrust × 0.20   // 0.50 → 0.30
spellW   = 0.30 - dnaTrust × 0.20   // 0.30 → 0.10
```

| sampleSize | DNA   | champWeight | spell+rune |
|-----------|-------|-------------|------------|
| 0판 (신규) | 20%   | 50%         | 30%        |
| 3판 (중간) | 44%   | 38%         | 18%        |
| 5판+ (충분) | 60%  | 30%         | 10%        |

- **신규/부캐**: 챔피언 통계 + 스펠/룬이 리드 → 챔피언의 기본 포지션에 수렴
- **충분한 전적**: DNA가 리드 → 유저의 실제 습관에 수렴
- **구현 범위 최소**: `scorePlayerForLane` 함수 내 가중치 계산 3줄 수정만으로 적용 가능

**4. Codex의 "하드락 강화"에 대한 반론**

정화/배리어 하드락은 이미 적용되어 있다. 추가 하드락(수호자 룬 → 서폿 하드락 등)은 오히려 위험하다. 예컨대 수호자를 드는 탑 플레이어(탑 사이온, 탑 탈리야)가 잘못 서폿으로 배정될 수 있다. **강한 소프트 신호(soft signal)가 여러 개 쌓이는 것이 단일 하드락보다 더 강건하다.**

**5. 위험 요소**

Adaptive 가중치를 도입하면 `scorePlayerForLane` 내부 공식이 바뀌므로, 안건 1에서 구현한 `runBacktrackFull`의 점수 체계도 같이 바뀐다. 5! 탐색이 올바른 결과를 내려면 상대적 순위만 맞으면 되므로 영향은 적지만, `isAnomalous` 임계값(현재 1.5)은 재검토가 필요할 수 있다.

### 3. 종합 및 심의
- **합의 영역**: '스펠 + 룬 + DNA'의 삼각 측량이 필수적임. 챔피언 이름만 보는 것은 한계가 있음.
- **해결**: **DNA(습관)에 60%, 챔피언 통계에 40%**의 가중치를 부여하여 사람 중심의 추론을 강화함.
- **Claude 추가**: 고정 가중치 대신 sampleSize 기반 적응형 가중치가 90→95% 갭 해소의 핵심이다. 추가 하드락보다 소프트 신호 누적 방식이 더 강건하다.

### 4. 최종 결정 및 근거 (The 95% Strategy)
- **결정**: **[Adaptive Signal Fusion + 확정적 닻 기반 계층적 배정 모델]** 도입.
- **근거**: 바텀은 '두 명'이 함께 가므로 서폿 닻만 찾으면 원딜이 확정되는 논리 + DNA 샘플 크기에 따라 신호 비중을 동적 조정하여 경계 구간 오판을 차단함.
